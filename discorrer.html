Pagina 4 – Discorrer sobre as fases do projeto

Etapa 1: Análise de requisitos Nesta primeira etapa é onde é analisado e trabalhado a origem do software, é nesta etapa que cada equipe considera cada ponto funcional do projeto ou solução para tal. É também neste momento que é feita a análise mais aprofundada dos requerimentos do usuário final, para que seja garantido e alcançado as expectativas do novo sistema. Etapa 2: Planejamento Como o nome já é auto explicativo, essa fase é o momento em que é feita a identificação do plano preliminar, para que seja feita a análise e se possa planejar os recursos fundamentais para a construção da infraestrutura do software. Recursos, capital, período, benefícios e outros itens devem ser levados em consideração nesta fase. Esta fase é determinada para que o escopo do projeto seja realizado e apresentado, para se ter a idéia do final do projeto. Etapa 3: Design e projeto Nesta etapa é de foco a todos discutirem e concretizar as demandas específicas para o sistema proposto. Será realizado nesta etapa a consideração necessária e essencial de hardware, software, estrutura, processamento e procedimento para o sistema atingir a expectativa proposta. Etapa 4: Codificação Nesta fase, é necessário que o desenvolvedor ou a equipe que está desenvolvendo o sistema do projeto siga certas diretrizes de codificação, é nesta fase que é realizado o uso de ferramentas de programação para gerar e implementar códigos. Etapa 5: Teste Assim que estiver finalizado o software, é realizado o chamado implantação no ambiente de teste. Os testes deste novo sistema começam a serem feitos e a equipe analisa cada detalhe para concluir se está tudo conforme a necessidade do cliente. É nesta fase que é feita toda uma análise e vistoria de possíveis bugs e erros, para que possa ser repassado ao desenvolvedor ou a equipe que está desenvolvendo o sistema. Etapa 6: Instalação Sendo finalizada a fase de testes, e confirmando um total de zero bugs ou erros, a fase de implantação é finalmente iniciada. Nesta fase é colocado em prova tudo de positivo retirado das fases anteriores, fazendo com que o caminho para a finalização do projeto fique mais claro e acessível. O software final é lançado e a verificação passa a ser nos problemas de implantação, caso haja. Etapa 7: Manutenção Nesta última fase, o foco principal é ter a certeza que o sistema permanecerá funcionando em perfeitas condições e as necessidades do cliente continuem a ser atendidas. Essa parte é feita a partir de resolução de bugs, atualizações para novas versões do sistema e do aprimoramento.
